Phase 1 테스트 가이드

1. 빌드 확인
- npm.cmd run build 실행 결과가 성공인지 확인합니다.

2. 공개 스토어 페이지 확인
- /store/[slug] 페이지를 엽니다.
- 스토어 이름, 위치, 프로모션 배너가 정상적으로 표시되는지 확인합니다.
- 프로모션 배너가 없는 스토어에서도 레이아웃이 깨지지 않는지 확인합니다.

3. 공개 연락 버튼 확인
- Store Admin에서 공개 연락 버튼이 활성화된 경우에만 버튼이 표시되는지 확인합니다.
- phone 타입은 tel: 링크로 동작하는지 확인합니다.
- messenger, facebook, link 타입은 새 탭으로 열리는지 확인합니다.
- owner_phone이 자동으로 공개 페이지에 노출되지 않는지 확인합니다.

4. 검색과 카테고리 필터 확인
- 검색어 입력 시 상품명이 기준으로 필터링되는지 확인합니다.
- 카테고리 탭을 선택한 상태에서 검색어를 입력해도 두 조건이 함께 적용되는지 확인합니다.
- All 탭 선택 시 전체 상품 기준으로 다시 표시되는지 확인합니다.

5. 상품 카드 확인
- 상품 이미지가 lazy loading으로 유지되는지 확인합니다.
- 이미지가 없는 상품은 No Image 영역이 표시되는지 확인합니다.
- FEATURED, BEST SELLER, promo label, COMBO 배지가 기존 조건대로 표시되는지 확인합니다.
- 품절 상품은 Out of Stock 오버레이와 품절 스타일이 유지되는지 확인합니다.
- 가격, 설명, 재고 상태가 카드 안에서 겹치지 않는지 확인합니다.

6. 정렬 확인
- 기존 상품 정렬 순서가 바뀌지 않았는지 확인합니다.
- 이번 변경은 클라이언트 표시 스타일 중심이며 서버 정렬 로직은 변경하지 않았습니다.

7. 모바일 확인
- 모바일 폭에서 검색 박스, 카테고리 탭, 상품 카드, 연락 버튼이 화면 밖으로 넘치지 않는지 확인합니다.
- 카테고리 탭은 가로 스크롤로 사용할 수 있는지 확인합니다.
